HTML 기초

  1. HTML은 Hyper Text Markup Language를 나타냅니다.

    하이퍼텍스트(Hyper Text)란?

    - 순서가 없는 정보를 담고 있는 텍스트


    마크업(Markup) 이란?

    - 정보를 구조화하여 순서대로 웹 페이지에 띄우는것


    Hyper Text Markup Language란?

    - 순서가 없는 정보를 담고 있는 텍스트의 정보를 구조화하여 순서대로 웹 페이지에 표시하기 위한 브라우저 언어


    [ 구조화란? ]

    - 구조 -> 체계 -> 순서/분류 -> 체계가 없는 데이터를 체계있게 순서를 정해주는 것

  2. HTML 은 XML에서 온 것이다!

    - XML 은 정보를 구조화하는 확장가능한 마크업 언어이다!


    표기법

    1) 최상단에 xml선언부
    2) 최상위 태그가 단 하나임! -> Root라고함!
    3) 하위 구조화태그 생성

    예시 보기

    <?xml version="1.0" encoding="UTF-8"?>
    <회사정보>
    <회사 창립일="1988년">
    <회사명>유아이 < /회사명>
    <사원수>3000 </사원수>
    <연매출>1억</연매출>
    </회사>
    <회사 창립일="2000년">
    <회사명 >데이터굿</회사명>
    <사원수>400</사원수>
    <연매출>10억</연매출>
    < /회사 >
    </ 회사정보 >


    참고: 파일확장자란?

    - 파일 포멧을 알려주는 표시자
    ※ 파일 확장자를 반드시 보이게할것!

  3. HTML의 태그구조

    - 태그: 구조화 요소의 표시자
    - 태그(Tag)는 요소(Element)라고도 함

  4. 태그구조

    1. 선언부

      <!doctype html>

      브라우저가 이것을 읽는 순간
      html5문법으로 읽을 준비를 한다

    2. 문서에서 html 태그는 : 오직한번만 사용한다

      이것을 다른말로 Root라고 함

    3. 머리(head)

      -> 웹 페이지 타이틀, 인코딩설정 등 메타데이터(요약데이터)

      meta태그: 문서의 기본정보

      - charset="utf-8"
      -> 한글, 일본어, 한자어 등 2byte 체계의 문자셋이 브라우저에 잘 표시되도록 인코딩 설정함!
      - 사이트의 기본 정보 등을 표시함

      title태그 : 브라우저 탭에 제목출력

    4. 도큐먼트 (body)

      브라우저에 출력되는 내용부분

      <body></body>

    5. 저장시 확장자

      확장자

      -> .html 또는 .htm
      -> 이것이 브라우저가 읽을 수 있는 웹문서 확장자
      - 이름이 같아도 확장자가 다르면 다른문서로 인식함
      예) main.html , main.htm 서로다른 웹문서임

      참고사항

      참고) 첫번째 홈페이지는 index.html 로 하는 것이 국룰이다!
      왜? 웹서버가 이름없이 가장먼저 읽는 기본 파일명이다!

    6. 코딩툴

      vs code, 에디트 플러스, 아톰, 서브라임텍스트3 , 메모장으로 하드코딩도 가능

    7. 요소간 관계

      - 어떤요소의 하위요소가 있다면 상위요소 "부모요소"
      하위요소는 "자식요소"라고 한다.

    8. html 주석

      - 출력되지 않는 구간표시

      <!-- 주석내용 -->

    9. 태그 구성요소

      <요소명 속성명="값" >
      참고사항

      - 요소명은 정해져 있다.
      - 정해진 속성명이 있고 만들 수 있는 속성명도 있다!
      새로운 속성을 만들려면 반드시 data-로 시작해야함! (임의의 변수를 설정할 수 있음)

    10. HTML 표기에 대한 기본원칙

      1. 1) 소문자로 표기한다.

      2. 2) 속성명의 값은 쌍따옴표를 사용한다.

      3. 3) 요소와 속성명 사이, 속성과 속성 사이는 반드시 한칸 띄어쓴다.

    11. html요소의 두 가지 구분법

      블록요소

      - 여러가지 요소를 담는 구분구역을 감싸는 요소
      - 내용상 구분을 목적으로 한다!
      - 특징: 블록요소는 설정라인 끝까지 잡으므로
      옆에 다른 요소가 올 수 없다!
      - 대표요소들: div, p, h1~h6, blockquote, ul, ol, dl 등
      - 블록요소 대표: div(인라인과 블록요소들을 담는 구역설정시 사용)

      인라인요소

      - 내용상 구분하는 구역설정요소
      - 블록요소 내부에 주로 위치함
      - 특징: 인라인요소는 옆에 다른 인라인요소가 올 수 있다!
      - 주의 사항: 인라인요소는 블록요소를 포함할 수 없다!
      - 대표요소들: span, a, em, b, i, bdo, q, strong 등
      - 인라인요소 대표: span(인라인요소 범위를 잡을 때 사용함)

    12. HTML 특수문자처리 (엠퍼샌드와 세미콜론사이에 표기)

      태그문자 꺽쇠 표시하기
      < & lt; 여는 꺽쇠는 ~보다 작다(less than)
      > & gt; 닫는 꺽쇠는 ~보다 크다(greater than)
      줄바꿈없는 공백표시하기(no-break space) - & nbsp;
      & 엔퍼샌드(ampersand) & amp;

  • 문자 데이터 관련 태그

    1. h1~h6 : 제목(Heading)

      요소레벨 블록 / 일반적으로 문서의 제목으로 사용함

    2. p : 단락(내용구분-Pharagraph)

      요소레벨 블록

    3. a : 링크 (Anchor:닻)

      링크 속성 설명

      - href 속성: 이동할 곳(hyper text reference)

      - target 속성: 창옵션(_blank새창)

      (기본값은 _self 현재창)

      기본적으로 폰트색상과, 텍스트 데코레이션에 밑줄을 가지고 있으며 초기화 해서 사용해야함

    4. br : 줄바꿈 (break )

    5. hr : 가로구분선 (horizontal 수평선)

      줄바꿈과 선을 표기할때 사용하나 디자인적으로 사용할때는 hr의 기본 선의 스타일이 있기때문에 이 점을 주의할것

  • HTML 텍스트 서식

    b - 굵은 텍스트 (bold 두꺼운체)

    strong - 중요한 텍스트 (의미-시멘틱) -

    B태그와 Strong태그의 차이점

    B태그는 시각적으로 굵게만 보이고 strong도 굵은 텍스트는 같지만

    b태그는 리더기에서 강조해주지 않고 strong태그는 리더기에서 강조를 해준다는점 !! (시멘틱)

    i - 기울임꼴 텍스트 (italic 기울림글꼴)

    em - 강조된 텍스트 (emphasize 강조하다 - 의미-시멘틱)

    i태그와 em태그의 차이점

    i태그와 em는 시각적으로 기울어져 보이는것은 같지만

    i태그는 리더기에서 강조해주지 않고 em태그는 리더기에서 강조를 해준다는점 !! (시멘틱)

    mark - 표시된 텍스트 (노란색 글자배경-눈에 잘띔)

    small - 더 작은 텍스트

    del - 삭제된 텍스트 (delete 지우다)

    ins - 삽입된 텍스트 (insert 삽입하다)

    sub - 아래 첨자 텍스트(subscript text)

    sup - 위 첨자 텍스트(superscript text)

  • 인용구관련 태그

    blockquote - 인용된 내용을

    블록처리

    - cite 속성: 인용된 사이트 주소(출처) - 선택옵션

    q - 짧은 인용구(인라인) - quotation 태그로 감싼부분에 " " 자동 표기됨

    abbr - 약자 (Abbriviation 줄임말)

    abbr

    - title속성을 사용하여 원래의 뜻을 툴팁으로 띄워준다!

    -> title은 abbr의 필수속성!

    (title속성은 어떤 요소에나 모두 사용가능하고 툴팁을 띄움)

    address - 주소,전화번호,대표자 등(유의미-시멘틱)

    -> 블록요소, 이탤릭 기본적용

    bdo - 글자진행방향 변경 (반대방향으로 쓰는 문자지원)

    - dir="rtl" -> 오른쪽에서 왼쪽으로 출력! -> 문자가 거꾸로 표시가능

  • 이미지관련 태그

    figure - 사진, 표, 도형 등을 담는 블록박스

    figcaption - figure요소의 하위 자식요소

    사용예시

    img - 이미지 표시요소
    - src 속성: 이미지호출 경로

    - alt 속성: 이미지 설명(웹 접근성-평등의원칙 필수!)

    또하나의 특징으로 width height 값을 가질 수 있고, text-align 으로 정렬도 가능하다.

    기본적으로 a 태그처럼 초기화 해야하는 값들이 존재함

  • 그룹 요소 (블록요소)

    ul : 순서없는 리스트(unordered list)
    -> 순서가 없는 리스트이기 때문에 블릿옵션을 가지고 있음 : list-style disc

    ol : 순서있는 리스트(ordered list)
    -> 순서가 있는 리스트이기 때문에 순서를 표시해줌 : list-style upper-greek

    li : ul과 ol요소의 필수 자식요소!

    dl : 용어설명리스트 (Description Lists)

    dt : 용어 (defines terms)

    dd : 용어설명 (defines description)

    -> dl 아래 자식으로 반드시 dt와 dd는 한개이상 와야함!
    -> dt와 dd는 형제요소임!

  • 의미 없는 빈 블록/인라인요소

    div : 내용을 구분할때 사용하는 요소

    특징

    -> display:block 라는 설정외에 아무런 셋팅이 없다
    -> 자유롭게 블록요소로 구조를 잡을때 사용함

    span : 인라인 내용요소나 문자데이터를 구분할때 사용하는 요소

  • 테이블

    table : 테이블 전체 부모요소

    tr : 테이블 레코드행 (table row)

    td : 테이블 데이터요소 (table data)

    th : 테이블 머릿부분에 사용될 경우
    td대신 th를 사용할 수 있다.
    (기본디자인-중앙정렬+두꺼운글자)

    (테이블 구조를 위한 추가요소들)

    caption : 테이블 제목

    thead : 테이블 머릿부분

    tbody : 테이블 중앙 내용부분

    tfoot : 테이블 하단부분

    테이블 실사용예제

    구조를 위한 추가요소들을 활용해서 thead tbody tfoot중에 리더기에 먼저 나와야 하는 부분의 경우 순서를 조정해서 사용이 가능하며
    리더기 사용자를 위해 th들에 id값을 지정한 후에
    td에서 해당 내용의 값이 출력될때
    headers = "먼저 불릴th 요소의 내용 , 뒤에 불릴 th요소 내용"
    을 사용해서 리더기 사용자들의 사용성을 증대 가능
    <th id="s" >삼성 </th>
    <th id ="t" >태블릿 </th>
    <td headers="s t">170,000</td> -> id s에 해당하는 행의 내용을 읽고
    id t에 해당하는 행의 내용을 읽어준뒤 td태그에 있는 값을 읽어줌
    해석하면 삼성 , 태블릿 170000 이라고 리더기에서 읽어줄것임

    중고폰
    구분 삼성 애플
    합계 370,000 390,000
    스마트폰 150,000 160,000
    태블릿 170,000 190,000

  • 모달창

    dialog 태그를 활용하여 조금더 쉽게 제작은 가능함

    태그는 기본적으로 네모난 박스형태이며 display : none상태로 시작됨

    dialog의 속성에 open이라는 속성에 true 값을 주면 display = block 상태로 전환됨

    기본값은 false상태로 스크립트에서 dialog.open = true or false 로 값을 변경함으로 손쉽게 모달창을 제작 가능함

  • 템플릿

    html의 템플릿 기능이며 html문서에서 작성시 해당 내용은 html문서에서 나타나지는 않음


    스크립트 등을 활용하여 반복적으로 사용해야하는 태그의 반복이 있는경우


    html에서 템플릿을 작성해두고 이를활용하여 스크립트상에서 템플릿에 내용을 작성후 원하는 태그에 prepend 나 append 로 추가하는 형태로 사용

  • details/summary

    일반적으로 버튼을 만들고 해당 버튼을 클릭하면

    내부에 있는 내용이 노출될 수 있도록 작업을 하는 경우에

    자바스크립트를 사용해서 관련 내용을 만든다.

    하지만 html5에서 해당기능을 지원해준다.

    details 태그는 html5에서 추가 된 태그로
    하위 요소로 summary > 태그를 사용해서 detail 태그 안의 내용을 클릭 할때마다 열기/닫기를 통해 보여준다.

    기본 속성으로 marker를 가지고 있으며

    이 속성은 css에서 details summary::marker{font-size:0;} 를 사용해서 제거가 가능하다

  • form태그

    - 입력된 데이터를 묶어서 서버로 전송하는 역할

    - action 속성 : 서버전송 페이지 주소

    -> get방식은 url로 데이터를 전달하는 방식

    -> post방식은 페이지로 숨겨서 데이터를 전달하는 방식

    여러라디오 버튼중 하나만 선택된다!

    4) checkbox - 체크박스버튼(보통 다중 선택시 사용)

    5) button - 버튼형식의 입력폼

  • CSS 변수

    CSS 변수는 바(-) 2개를 붙여 표시합니다 --red: #a00;

    "--red" 변수명에 "#a00" 이라는 속성 값이 적용된 것입니다.

    .content{ --newcolor: #000; } 이렇게 변수를 선언했다면 content라는 클래스의 하위 요소에서만

    해당 변수를 사용 할 수 있습니다.

    전역으로 사용하기 위한 변수(들)은 다음과 같이 ":root" 가상 클래스에 정의합니다.

    :root{ --red: #a00; --dark: #404040; --light: #c0c0c0; }

    변수의 사용

    선언한 CSS 변수를 사용시 변수 값을 적용할 속성 값 위치에 "var()"로 변수명을 감싸서 표시하면 됩니다.

    color: var(--red);

    자바스크립트에서 변수의 접근

    let theme = document.querySelector(':root');

    let styles = getComputedStyle(theme);

    console.log(styles.getPropertyValue('--hover')); // 변수 값 얻기

    theme.style.setProperty('--hover', 'green'); // 변수 값 변경

    단 자바스크립트로 변수를 제어시 html태그에 인라인 형식으로 직접 적용되기 때문에 우선순위를 기억할 것

  • CSSOM (CSS Object Model)

    HTML과 마찬가지로 CSS도 브라우저가 이해할 수 있는 형식으로 변환해야 합니다.

    HTML 마크업 내에 직접(인라인) 스타일을 선언하지 않고 CSS와 HTML을 별도로

    유지하면 콘텐츠와 디자인을 별도의 항목으로 처리할 수 있습니다.

    CSS 객체 모델은 브라우저가 DOM을 생성하는 프로세스와 같습니다.

    따라서 HTML 대신 CSS에 대해 DOM 프로세스를 진행하며 CSSOM(CSS Object Mode)도

    트리 구조를 갖습니다.

    CSSOM이 트리 구조를 갖게 되면서 cascading 룰을 갖고 스타일을 세분화합니다.

    DOM과 CSSOM은 독립적인 데이터 구조입니다

  • CSS 랜더링 순서**

    브라우저에서 html파싱 및 dom트리를 형성하고 css도 파싱 후 cssom을 형성하여 렌더링 트리를 형성한다

    렌더링 트리에서 레이아웃을 실행하여 각 노드의 기하학적 형태를 계산한다.

    레이아웃 단계

    - 레이아웃 단계에서 노드의 정확한 위치와 크기를 계산한다.

    - 노드의 정확한 크기와 위치를 파악하기 위해 루트부터 노드를 순회하면서 계산하고,

    레이아웃 결과로 각 노드의 위치와 크기를 픽셀 값으로 렌더트리에 반영한다.(ex: % -> px)

    개별 노드를 화면에 페인트한다

    페인트 단계

    픽셀을 채우는 프로세스로써 텍스트 색상, 이미지, border, shadow 등 요소의 시작적인 부분을 그리는 작업이 포함된다.

    - 이전 레이아웃 단계에서 계산된 값을 이용해 렌더트리의 각 노드를 화면상의 실제 픽셀로 변환되는데 페인팅 단계에서는

    CSS 속성(색상, 투명도 등)을 적용한다.

    - 픽셀로 변환된 결과는 개별 레이어로 관리된다.

    합성 & 렌더 (composite)

    composite

    페이지가 정확히 렌더링되려면 정확한 순서로 화면을 그려야한다.

    - 페인트 단계에서 생성된 레이어를 합성하여 스크린을 업데이트하고 합성과 렌더링이 끝나면 화면에서 웹 페이지를 볼 수 있다.

    레이아웃과 리페인트

    렌더트리는 자바스크립트에 의해 DOM 트리, CSSOM 트리가 변경될 때 다시 재구성된다.

    스타일 -> 레이아웃 -> 페인트 -> 합성을 렌더링이라고 하고 상황에 따라 반복하여 발생할 수 있다

    DOM이 추가, 삭제되거나 높이, 넓이, 위치를 주는 CSS 속성값이 변경되는 경우 렌더트리가 재구된다.

    (배경색, 폰트색, 텍스트 데코레이션)레이아웃 과정을 건너뛰고 페인트부터 수행하는데 이를 리페인트.

    레이아웃부터 이후 과정을 수행하는 것을 레이아웃 또는 리플로우라고 한다.

  • 미디어쿼리란

    디바이스의 환경에 따라

    사전에 설정해둔 값에 맞춰서 반응형으로 홈페이지를 표현해 줄 수 있는

    코딩 기법

    반응형 웹 디자인(Responsive Web Design)의 핵심 요소

    미디어 유형은 장치의 종류를 나타내며, 예를 들어 스크린(screen), 프린터(printer), 스피커(speech) 등이 있습니다

  • 왜 사용하는가

    넓은 화면에서 보이던 레이아웃이 작은화면에

    맞지 않는 가독성과 구조의 복잡함을 해결하기위해

    각 사이즈별로 디자인을 변경할 수 있도록

    미디어쿼리를 사용이 가능하며

    상황에 맞게 다크 모드를 사용중인 사람이라면 다크모드로 표현도 가능하며

    홈페이지를 사용하는 사용자에게 조금더 사용성이 좋게 디자인된 페이지를 제공할 수 있다.

  • 유형 및 종류

    미디어 유형에는 all(모든 매체), print(프린터 기기), screen(컴퓨터, 태블릿, 스마트폰 등), speech(스크린 리더) 등이 있다.

    이 중 all, screen, print를 가장 많이 쓰이는데 생략이 가능해서 생략하셔도 되며. 생략할 경우 기본값은 all 이다.

    미디어 유형 설명
    all 모든 미디어 유형
    screen 컴퓨터 스크린(스마트폰 스크린 포함) - 가장 많이 사용
    print 인쇄 장치
    tv 음성과 영상이 동시 출력되는 TV
    aural 음성 합성 장치(주로 화면을 읽어 소리로 출력해주는 장치)
    braille 점자 표시 장치
    handheld 패드(pad)처럼 손에 들고 다니는 장치
    projection 프로젝터
    tty 디스플레이 기능이 제한된 장치(픽셀(px) 단위를 사용할 수 없음)
    embossed 점자 프린터

  • 작성방법

    미디어 쿼리는 CSS의 @media 규칙을 사용하여 정의해야 한다.

    @media only screen and (max-width:800px){ 변경할 CSS 코딩 }

    위 코드 해석

    화면 너비가 800px 이하인 경우에만 적용되는 스타일

    해당 쿼리 내용으로는 해당 사이즈에서 변경되는 내용만 작성하면 된다. 이미 적용되어 있는 값은 따로 적을 필요 없음

    작성시 주의사항은 CSS에서 미디어 쿼리의 작성 위치도 중요하다

    breakpoints를 높은 값에서 낮은 값 순으로 내려가면서 작성해야하며

    순서를 어기게 되면 미디어쿼리 동작이 원활하게 이루어 지지 않을 수 있음



    breakpoints를 생각해서 쿼리를 작성해야함

  • 연산자

    연산자 설명
    and 앞의 소스처럼 조건을 계속 추가할 수 있습니다.
    ,(쉼표) 동일한 스타일 유형을 사용할 미디어의 유형과 조건이 있다면 쉼표를 이용하여 추가합니다.
    only 미디어 쿼리를 지원하는 웹 브라우저에서만 조건을 인식하게 합니다. 이 키워드를 사용하면 미디어 쿼리를 지원하지 않는 웹 브라우저에서는 미디어 쿼리를 무시하고 실행하지 않습니다.
    not not 다음에 지정하는 미디어 유형을 제외합니다. 예를 들어 'not tv'라고 지정한다면 TVz를 제외한 나머지 미디어 유형에만 적용됩니다.

    자바스크립트 숫자 모든 JavaScript 숫자는 10진수(부동 소수점)로 저장됩니다. 숫자는 소수로 또는 소수로 쓸 수 없습니다. 대부분의 프로그래밍 언어에는 다음과 같은 많은 숫자 유형이 있습니다. 정수(정수): byte(8비트), short(16비트), int(32비트), long(64비트) 실수(부동 소수점): float(32비트), double(64비트).

  • 3. 비긴트

    자바스크립트 BigInt 모든 JavaScript 숫자는 64비트 부동 소수점 형식으로 저장됩니다. JavaScript BigInt는 표현하기에는 너무 큰 정수 값을 저장하는 데 사용할 수 있는 새로운 데이터 유형(ES2020)입니다

    두 개의 값만 가질 수 있습니다.true false

    JavaScript에서 값이 없는 변수에는 값이 있습니다. 유형도 입니다

  • 6. 널

    빈 값은 와 아무 관련이 없습니다.undefined 빈 문자열에는 올바른 값과 형식이 모두 있습니다.

  • 7. 배열

    JavaScript 배열은 대괄호로 작성됩니다. 배열 항목은 쉼표로 구분됩니다.
    배열 인덱스는 0부터 시작하며, 이는 첫 번째 항목이 [0]이고 두 번째 항목이 [1] 등입니다.

    JavaScript 객체는 중괄호로 작성됩니다.{}

    - const a = ["aa","bb","cc"]
